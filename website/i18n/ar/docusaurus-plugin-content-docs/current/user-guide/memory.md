# الذاكرة

تستخدم EstaCoda ملفات ذاكرة محدودة ومنظمة تستمر عبر الجلسات. الذاكرة هي سياق تنفيذ دائم، لكنها تابعة لتعليمات النظام، وتعليمات المطور، وسياق المستودع، و `AGENTS.md`، وسياسة الأمان، وطلب المستخدم الحالي.

تشرح هذه الصفحة ما هي ملفات الذاكرة الموجودة، وكيف تُملأ، وأين الحدود.

---

## ملفات الذاكرة

تعيش الذاكرة في نطاقين: محلية للملف الشخصي، ومشتركة عالميًا.

### الذاكرة المحلية للملف الشخصي

`~/.estacoda/profiles/<id>/` تحتوي على:

| الملف | الغرض | حد الأحرف |
|---|---|---|
| `USER.md` | تفضيلات المستخدم وأسلوب التواصل | 1,375 (~500 رمز) |
| `SOUL.md` | هوية الوكيل وشخصيته | قابل للتكوين |
| `MEMORY.md` | حقائق، واتفاقيات، ودروس | 2,200 (~800 رمز) |
| `promotions.json` | بيانات تعريف الترقية | — |

`AGENTS.md` **ليس** ملف ذاكرة. إنه سياق المشروع المُحمّل من مساحة العمل. لا يُنظّم، ولا يُضغط، ولا يُرقّى، ولا يُستدعى كذاكرة مكتسبة.

ترتيب العرض في الموجه:

```text
memory/shared/ -> USER.md -> SOUL.md -> MEMORY.md
```

### الذاكرة المشتركة العامة

`~/.estacoda/memory/shared/` تحتفظ بمقتطفات متاحة لجميع الملفات الشخصية. هي محدودة بواسطة المُصيّر وتحمل قبل الذاكرة المحلية للملف الشخصي.

## الاسترجاع المحلي اللفظي

الاسترجاع المحلي اللفظي منفّذ كقراءة/بحث حتمي فوق ملفات الذاكرة صاحبة السلطة، وليس استذكارًا دلاليًا، أو بحث vectors، أو بحث جلسات، أو طبقة سلطة ذاكرة جديدة.

الأسطح المنفّذة للمستخدم/المشغل:

| السطح | الغرض |
|---|---|
| `memory.read` | قراءة محتوى ذاكرة محلي محدود حسب مصدر/نوع الذاكرة |
| `memory.search` | بحث لفظي في محتوى الذاكرة المحلي |
| `estacoda memory index path` | عرض مسار الفهرس تحت حالة الملف الشخصي |
| `estacoda memory index status` | فحص مسار الفهرس المحلي، وصحته، وحالة pending rebuild |
| `estacoda memory index rebuild` | إعادة بناء صريحة لفهرس الذاكرة المحلي اللفظي |
| `estacoda memory search <query>` | بحث في الذاكرة المحلية من CLI |
| `estacoda memory read <source>` | قراءة الذاكرة المحلية من CLI |

الفهرس المحلي اللفظي مرآة قابلة لإعادة البناء ومخزنة تحت حالة الملف الشخصي:

```text
<profile-state-dir>/memory-index.sqlite
```

حذف ملف SQLite هذا لا يحذف `USER.md`، أو `SOUL.md`، أو `MEMORY.md`، أو ملفات الذاكرة المشتركة، أو `promotions.json`. أوامر حالة الفهرس وإعادة بنائه هي مسارات إصلاح للمشغل للمرآة. إذا كان الفهرس معطّلًا، أو مفقودًا، أو غير متاح، يستخدم `memory.read` و `memory.search` وقراءة/بحث CLI fallback آمنًا لقراءة الملف المباشرة أو بحث substring حيثما أمكن مع الحفاظ على ترشيح الذاكرة المحمية.

حدود مصادر السلطة:

- تبقى `USER.md`، و `SOUL.md`، و `MEMORY.md` ملفات مصدر محلية للملف الشخصي.
- تبقى ملفات الذاكرة المشتركة صاحبة السلطة تحت الذاكرة المشتركة العالمية.
- يبقى `promotions.json` صاحب السلطة لبيانات تعريف الترقية.
- `AGENTS.md` ليس ذاكرة ولا يُفهرس أبدًا كذاكرة.
- يمكن أن تُعكس الذاكرة المشتركة للاسترجاع اللفظي، لكن يجب أن يحفظ الفهرس حد مصدرها المشترك/العالمي.

ذاكرة الهوية/السلامة المحمية تبقى محمية. يُفهرس `SOUL.md` كمحمي لأجل parity، وحالة الفهرس، وفحوص إعادة البناء، لكن يستبعده `memory.read` و `memory.search` وقراءة/بحث CLI ما لم تكن `includeProtected` أو `--include-protected` صريحة. تبقى المقتطفات المحمية محدودة. يجب ألا يستخدم الاستذكار الدلالي مدخلات الهوية/السلامة المحمية أبدًا، وتبقى المدخلات المحمية مستبعدة من مسارات الاسترجاع المواجهة للدلالات.

يقبل `memory.read` و `memory.search` المفتاح `maxChars` لأنهما يعيدان محتوى الذاكرة مباشرة؛ وتحدّه الخدمة داخليًا. يقبل `memory.search` المفتاح `maxResults` المحدود. يجب ألا يقبل `session_search` المفتاح `maxChars`؛ تبقى حدود حجم النص فيه محكومة داخليًا من النظام. محتوى الذاكرة المُعاد منقّح من الأسرار، وموسوم بالمصدر، ومعلّم كسياق ذاكرة محلي، وليس تعليمات أعلى أولوية. المصادر المفقودة والنتائج الفارغة تعيد diagnostics منظمة.

تدعم قراءة CLI:

- `USER.md`
- `MEMORY.md`
- `SOUL.md` فقط مع `--include-protected`
- الذاكرة `shared` حسب المفتاح

مسار الاسترداد للمشغل:

```bash
estacoda memory index path
estacoda memory index status
# أوقف runtime قبل حذف ملف الفهرس.
rm <profile-state-dir>/memory-index.sqlite
estacoda memory index status
estacoda memory index rebuild
estacoda memory index status
```

الفهارس المفقودة تعرض تشخيص pending rebuild أو empty index. يمكن أن يعيد bounded startup backfill إنشاء الملف. إعادة البناء الكاملة صريحة عبر `estacoda memory index rebuild`، وهي idempotent، وتعيد التعبئة من ملفات الذاكرة صاحبة السلطة، وتفهرس `SOUL.md` كمحمي.

## الذاكرة المكتسبة الموثوقة مقابل الاستذكار غير الموثوق

ليست كل الذاكرة مُعاملة بالطريقة نفسها.

**الذاكرة المكتسبة الموثوقة** — `USER.md`، `SOUL.md`، `MEMORY.md`، والذاكرة المشتركة. هذه ملفات يتحكم فيها المشغل. تُحقن كسياق نظام.

**السياق التاريخي غير الموثوق** — استذكار الجلسة، والاستذكار الخارجي، وملخصات الضغط الدلالي. هذه مرجعية فقط. تُصنّف كتاريخية ولا يمكنها تجاوز تعليمات النظام، أو `AGENTS.md`، أو سياسة الأمان، أو الذاكرة المحلية، أو طلب المستخدم الحالي.

ترتيب السلطة:

```text
system/developer/repo/AGENTS/security/current user instructions > learned memory > reference-only recall/compression context
```

## سلامة كتابة الذاكرة

كتابات الذاكرة ليست غير مشروطة. يفحص النظام:

- **أنماط حقن الموجهات** — محظورة قبل الكتابة.
- **علامات الأسرار/مفاتيح API** — النص الذي يبدو كاعتمادات مرفوض.
- **ضوابط غير مرئية/ثنائية الاتجاه** — أحرف تحكم مشوهة أو مشبوهة يتم تطهيرها أو رفضها.

إذا فشلت الكتابة في فحص السلامة، تُرفض ولا يُعدّل الملف. تجاوز الترقية بعد رد مساعد ناجح بخلاف ذلك هو غير قاتل للدور؛ يُسجل تشخيص بأقصى جهد دون نص ترقي خام أو أسرار.

تستخدم كتابات الذاكرة المحلية أيضًا persistence واعيًا بالانحراف. قبل overwrite، يقارن مسار الحفظ ملف القرص الحالي مع snapshot المحمّلة سابقًا. تتضمن snapshot المسار، ونوع الذاكرة، و `mtimeMs`، والحجم، وcontent hash. التعديلات الخارجية تفشل مغلقة افتراضيًا، ورفض الانحراف يحافظ على ملف القرص الحالي. لا تُنشأ نسخ `.bak.<timestamp>` افتراضيًا؛ تُنشأ فقط عندما تفعّلها operation policy صراحةً. تشخيصات الانحراف لا تكشف محتوى الذاكرة الخام.

## الترقية

بعد كل دور، يفحص `memory-promotion.ts` تاريخ الجلسة المحدود بحثًا عن أنماط متكررة:

| النمط | الوجهة |
|---|---|
| تفضيلات مستخدم متكررة | `USER.md` |
| حقائق مشروع متكررة | `MEMORY.md` |
| نتائج المهارات | مخزن الذاكرة |
| استنتاجات يدوية | مخزن الذاكرة |

تتعامل الترقية مع التناقضات (استبدال المدخلات القديمة)، والتقوية (تعزيز المدخلات الموجودة)، والنسيان (إزالة المدخلات العالقة). إذا فشلت كتابة Markdown بعد تغييرات بيانات تعريف الترقية، يتراجع `LocalMemoryProvider` عن محتوى Markdown و `promotions.json`.

التعزيز **ليس** إصلاحًا ذاتيًا تلقائيًا. هو محدود، ومفحوص، ويفشل مغلقًا.

## الذاكرة الخارجية

الذاكرة الخارجية **معطّلة افتراضيًا**. المزود المنفّذ الوحيد هو مبني على الملفات ومحلي للملف الشخصي.

عند التمكين تحت `externalMemory`، تخزّن سجلات JSONL تحت دليل `external-memory/` الخاص بالملف الشخصي المختار. يمكنها:

- إرجاع استذكار خارجي محدود غير موثوق لدورات استذكار صريحة.
- مرآة كتابات `memory.curate` عندما يكون `mirrorWrites: true`.

لا يمكنها استبدال `USER.md`، أو `MEMORY.md`، أو `SOUL.md`، أو الذاكرة المشتركة، أو استذكار الجلسة. لا يمكنها استخدام مسارات تخزين مطلقة أو الكتابة خارج دليل `external-memory/` الخاص بالملف الشخصي.

أحداث التدقيق للاستذكار الخارجي وكتابات المرآة هي بيانات تعريف فقط. لا تخزّن أبدًا محتوى استذكار خام، أو محتوى مرآة خام، أو اعتمادات، أو أسرار.

## ضغط الجلسة

الضغط الدلالي للجلسة هو **تجريبي فقط** في v0.1.0. هو **معطّل افتراضيًا** ويتطلب كلا البوابتين:

```json
{
  "compression": {
    "enabled": true,
    "experimental": true
  }
}
```

ما لم يكن كل من `compression.enabled` و `compression.experimental` هما `true`، لا يعمل الضغط الدلالي. لا يوجد مسار افتراضي مُفعّل.

عند التمكين، يحوّل الضغط تاريخ الجلسة الأقدم إلى ملخصات مرجعية. لا يضغط `USER.md`، أو `SOUL.md`، أو `MEMORY.md`، أو `AGENTS.md`، أو الذاكرة المشتركة، أو بيانات تعريف الترقية. الملخصات المضغوطة هي سياق تاريخي غير موثوق.

الضغط اليدوي متاح عبر `/compact [topic]` أو `estacoda sessions compact <session-id>`. `/compact` في البوابة يحفظ نص الأب بإنشاء جلسة فرعية مضغوطة. `/compact` في CLI يبقى غير دوّار في هذا التنفيذ.

---

## ضغط ملف الذاكرة

ضغط ملف الذاكرة يضغط `USER.md` و `MEMORY.md`. لا يضغط `SOUL.md`، أو `AGENTS.md`، أو الذاكرة المشتركة، أو تاريخ الجلسة. يستخدم المسار الإضافي `memory_compaction`.

الأدوات:

- `memory.file_compact` — ضغط ملف يدويًا؛ يدعم `dryRun`.
- `memory.file_compaction_restore` — استعادة من نسخة احتياطية للضغط.

الضغط المُطبّق ينشئ نسخة احتياطية مُوقّتة تحت `.memory-file-compaction-backups/` قبل الكتابة. الضغط الحرج لملف الذاكرة هو تشخيصي فقط؛ لا يُشغّل ضغطًا تلقائيًا.

---

## أداة تنظيم الذاكرة

سطح كتابة الذاكرة الموجه للوكيل هو `memory.curate`. تقبل:

| النوع | الإجراء |
|---|---|
| `append` | إلحاق مدخل جديد |
| `replace` | استبدال مدخل موجود بالمطابقة الجزئية |
| `remove` | إزالة مدخل بالمطابقة الجزئية |

يستخدم الاسترجاع المحلي اللفظي أداتي `memory.read` و `memory.search` منفصلتين بدل إضافة سلوك قراءة إلى `memory.curate`. هاتان الأداتان تقرآن/تبحثان سياق الذاكرة المحلي فقط. لا تكتبان ذاكرة، ولا ترقيان محتوى، ولا تغيّران أولوية التعليمات.

## الفحص، والنسخ الاحتياطي، والاسترداد

```bash
# فحص وإصلاح الفهرس المحلي اللفظي
estacoda memory index path
estacoda memory index status
estacoda memory index rebuild

# بحث/قراءة الذاكرة المحلية من CLI
estacoda memory search <query>
estacoda memory read USER.md
estacoda memory read MEMORY.md
estacoda memory read shared <key>
estacoda memory read SOUL.md --include-protected

# فحص ضغط ميزانية الذاكرة
# (يظهر في التشخيصات؛ لا يوجد CLI مستقل في v0.1.0)

# ضغط ملف ذاكرة (عبر أداة وقت التشغيل)
# memory.file_compact target=USER.md dryRun=true

# استعادة من نسخة احتياطية (عبر أداة وقت التشغيل)
# memory.file_compaction_restore target=USER.md
```

ملفات الذاكرة هي Markdown عادي. يمكنك نسخها احتياطيًا يدويًا:

```bash
cp ~/.estacoda/profiles/<id>/USER.md ~/.estacoda/profiles/<id>/USER.md.bak
cp ~/.estacoda/profiles/<id>/MEMORY.md ~/.estacoda/profiles/<id>/MEMORY.md.bak
```

لا تُعدّل `promotions.json` بعمى. يتتبع بيانات تعريف الترقية النشطة. إذا انحرف عن ملفات Markdown، قد يُعيد عرض الذاكرة إدخالات قديمة أو مرفوضة.

## أوضاع الفشل

**تجاوز الترقية:** غير قاتل للدور. يُسجل تشخيص ببيانات ضغط فقط. لا يتضمن نصًا خامًا أو أسرار.

**رفض الماسح:** الكتابة محظورة. الملف الأصلي محفوظ. لا تُترك بيانات تعريف ترقية نشطة.

**فشل ضغط ملف الذاكرة:** الملف الأصلي محفوظ. المسار الإضافي `memory_compaction` المفقود يُعيد `memory-file-compaction-route-unavailable`.

**فشل الذاكرة الخارجية:** الذاكرة المحلية تبقى صاحبة السلطة. تظهر الأعطال كتحذيرات.
