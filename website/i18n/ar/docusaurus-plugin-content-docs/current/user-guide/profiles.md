# الملفات الشخصية

الملفات الشخصية هي حدود تشغيلية، وليست تفضيلات تجميلية. يعزل كل ملف شخصي الإعدادات والاعتمادات والذاكرة والمهارات وحالة cron وحالة البوابة والسجلات. تبديل الملف الشخصي يعني تبديل سياق التشغيل بأكمله.

توضح هذه الصفحة كيفية عمل الملفات الشخصية، وأين تقع حالتها، وما يحدث عند اختيار الملف الشخصي الخاطئ.

---

## ما هي الملفات الشخصية

الملف الشخصي هو دليل تحت `~/.estacoda/profiles/<profile-id>/` يحتوي على سياق تشغيل كامل ومستقل. عند اختيار ملف شخصي، تحمل EstaCoda إعداده واعتماداته وذاكرته ومهاراته وقاعدة بيانات جلساته. لا شيء يتسرب بين الملفات الشخصية.

الملفات الشخصية للاستخدام متعدد السياقات: مزودون مختلفون، ومساحات عمل مختلفة، وهويات مختلفة، ومواقف أمان مختلفة. إنها ليست سمات خفيفة الوزن.

---

## المسارات المحلية للملف الشخصي

```
~/.estacoda/profiles/<profile-id>/
  config.json              # إعدادات التشغيل
  .env                     # متغيرات بيئة الاعتمادات (0600)
  auth.json                # رموز OAuth وحالة المصادقة
  USER.md                  # تفضيلات المستخدم والأسلوب
  SOUL.md                  # هوية الوكيل
  MEMORY.md                # حقائق واتفاقيات ودروس
  promotions.json          # بيانات تعريف الترقية
  skills/                  # المهارات المحلية للملف الشخصي
  cron/                    # حالة مهام cron
  gateway/                 # حالة البوابة والسجلات
    logs/
    state/
  external-memory/         # ذاكرة خارجية مستندة إلى الملف
  .memory-file-compaction-backups/  # نسخ احتياطية للضغط
```

الملف الشخصي النشط مسجل في:

```
~/.estacoda/active-profile.json
```

## أوامر الملف الشخصي

```bash
# إنشاء ملف شخصي
estacoda profile create <name>

# إنشاء ملف شخصي فارغ
estacoda profile create <name> --blank

# نسخ ملفات ذاكرة من ملف شخصي آخر
estacoda profile create <name> --from <profile> --files user,memory,soul

# إدراج الملفات الشخصية
estacoda profile list

# تبديل الملف الشخصي النشط
estacoda profile use <name>

# عرض تفاصيل الملف الشخصي (يخفي الأسرار)
estacoda profile show [name]

# حذف ملف شخصي
estacoda profile delete <name>

# إعادة تسمية ملف شخصي
estacoda profile rename <old> <new>
```

السلوك:

- `profile create` ينسخ `USER.md` و `MEMORY.md` من الملف الشخصي النشط افتراضيًا وينشئ `SOUL.md` فارغًا جديدًا.
- `profile create --blank` ينشئ ملفات ذاكرة فارغة.
- `profile use` هو الأمر العادي الوحيد الذي يحدث `active-profile.json`.
- `profile show` يبلغ عن المسارات وملخص النموذج مع إخفاء القيم السرية.
- `profile delete` يرفض الملفات الشخصية النشطة أو غير الفارغة ما لم يتم توفير `--force`.
- `profile rename` يحدث سجل الملف الشخصي النشط عندما يكون الملف الشخصي المُعاد تسميته هو النشط.

## اختيار الملف الشخصي لمرة واحدة

```bash
estacoda --profile work model status
estacoda -p work doctor
```

`--profile` / `-p` يختار ملفًا شخصيًا للأمر الحالي فقط. لا يغير `active-profile.json`. هذا مفيد للأوامر الفردية ضد ملف شخصي غير نشط دون تبديل السياق.

---

## ما يملكه الملف الشخصي

| الحالة | الموقع | النطاق |
|---|---|---|
| إعداد المزود | `config.json` | الملف الشخصي |
| الاعتمادات | `.env`، `auth.json` | الملف الشخصي |
| ملفات الذاكرة | `USER.md`، `SOUL.md`، `MEMORY.md` | الملف الشخصي |
| المهارات | `skills/` | الملف الشخصي |
| مهام cron | `cron/` | الملف الشخصي |
| حالة البوابة | `gateway/` | الملف الشخصي |
| صفوف الجلسة ومهام الإنهاء | `~/.estacoda/sessions.sqlite` | قاعدة بيانات عامة، معزولة بواسطة `profile_id` |
| الذاكرة الخارجية | `external-memory/` | الملف الشخصي |
| منح الثقة | `~/.estacoda/trust.json` | عالمي (مملوك للدليل، وليس للملف الشخصي) |

ثقة مساحة العمل هي حالة عالمية مملوكة للدليل، وليست خاصة بالملف الشخصي. مساحة عمل موثوقة تكون موثوقة عبر الملفات الشخصية. الإعدادات والاعتمادات والذاكرة لا تُشارك أبدًا.

## أوضاع الفشل

**ملف شخصي نشط خاطئ:** إذا نفذت أمرًا واستخدم المزود أو النموذج أو الاعتمادات الخاطئة، تحقق من الملف الشخصي النشط باستخدام `estacoda profile show`. قم بالتبديل باستخدام `estacoda profile use <name>`.

**متغير بيئة مفقود:** إذا أبلغ مسار المزود عن اعتماد مفقود، تأكد من ضبط متغير البيئة في ملف `.env` الخاص بالملف الشخصي أو تصديره في الصدفة قبل التشغيل.

**إعداد مفقود:** إذا كان `config.json` مفقودًا أو غير قابل للتحليل، يتراجع التشغيل إلى الإعدادات الافتراضية أو يوجه إلى الإعداد. شغّل `estacoda verify` للتحقق من صحة الإعداد.

**البوابة تعمل ضد ملف شخصي آخر:** تكون البوابة مرتبطة بالملف الشخصي المختار عند بدء التشغيل. تغيير `active-profile.json` لا يؤثر على بوابة قيد التشغيل. إذا بدأت البوابة ضد الملف الشخصي `work` ثم قمت بالتبديل إلى `personal`، تستمر البوابة في خدمة `work`. أوقف وأعد تشغيل البوابة بالملف الشخصي المطلوب.

**حظر حذف الملف الشخصي:** يرفض `profile delete` حذف الملف الشخصي النشط أو الملف الشخصي ذي الحالة غير الفارغة ما لم يتم توفير `--force`. هذا يمنع الفقدان العرضي للجلسات أو مهام cron أو سجلات البوابة.

## الفحص والاسترداد

```bash
# عرض تفاصيل الملف الشخصي النشط
estacoda profile show

# إدراج جميع الملفات الشخصية
estacoda profile list

# التحقق من الإعداد للملف الشخصي النشط
estacoda verify

# فحص المزود المباشر
estacoda doctor --live

# التحقق من أن البوابة مرتبطة بالملف الشخصي المتوقع
estacoda gateway status
```
