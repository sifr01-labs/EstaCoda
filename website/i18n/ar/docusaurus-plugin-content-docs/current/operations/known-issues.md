# المشكلات المعروفة

هذه الصفحة مقصودة أن تكون صريحة. هي لاستمرارية الهندسة، وليست للتسويق. كل قيد مدرج هنا هو حد، وليس وعدًا.

## غير مدعوم في v0.1.0

هذه الأسطح موجودة في الكود أو السجل لكنها ليست حية للمستخدمين.

| السطح | الحالة | التفصيل |
|-------|--------|---------|
| مثبت Windows الأصلي | غير مدعوم | لا يوجد مسار تثبيت أصلي لـ Windows. WSL أفضل جهد. |
| تحديث بإثر فقط عبر `ESTACODA_UPDATE_ARTIFACT` | متاح، غير مُستحسن | المسار الفني يوجد في الكود لكنه ليس الميكانيسم العام للتحديث في v0.1.0. استخدم `estacoda update` أو توجيه مدير الحزم. |
| مزودو المتصفح السحابي ما عدا Browserbase | مسجلون، غير حيين | browser-use وFirecrawl وCamofox مسجلون لكن لا يمكنهم إنشاء جلسات حية. Browserbase مُنفّذ خلف موافقة صريحة على الإنفاق. |
| مزودو بحث الويب بخلاف Brave وDDGS وfetch | مسجلون، غير حيين | Firecrawl وParallel وTavily وExa وSearXNG أطر وهمية. يوفر Brave وDDGS بحثًا حيًا، و`fetch` المدمج الحارس حي للاستخراج. |
| محول Anthropic Messages API | معروف بالكتالوج، غير قابل للتشغيل | الكود موجود لكنه غير قابل للتشغيل في البناء الحالي. |
| محول MiniMax LLM | معروف بالكتالوج، غير قابل للتشغيل | يظهر في البيانات الوصفية لكنه ليس مسار استدلال حي. |
| محول Nous LLM | معروف بالكتالوج، غير قابل للتشغيل | يظهر في البيانات الوصفية لكنه ليس مسار استدلال حي. |
| مزودو ذاكرة خارجية عشوائية | غير مدعومون | فقط موفر `file` المدمج يبني موفرًا حيًا. يُرفض الموفرون المسَمّون بدون تطبيق كود. |

## تجريبي في v0.1.0

هذه الميزات محصورة بالكود أو مُعلّمة بالنضج. فعّلها فقط إذا فهمت المخاطر.

| الميزة | البوابة | المخاطر |
|--------|---------|---------|
| ضغط الجلسة | `compression.enabled` و`compression.experimental` كلاهما `true` | تجريبي فقط. معطل افتراضيًا. |
| استقلالية Agent Evolution فوق `suggest` | أوضاع `skills.autonomy` فوق `suggest` | يسجّل أدلة وproposal قابلة للمراجعة وقرارات في وضع ظل فقط. الترقية التلقائية، والاسترجاع التلقائي، وإنشاء المهارات المحلية تلقائيًا ليست مفعّلة. |
| سير عمل تطور/مقترح/ترقية المهارات | `skill.propose_patch`، `skill.rollback` | محكوم لكن ليس مستقلًا بالكامل. الترقية تُشغّل بوابات eval. |
| تكامل وقت تشغيل Workflow | يتطلب قاعدة بيانات جلسات SQLite | أفضل جهد. يتصل فقط عندما تكون قاعدة بيانات الجلسة SQLite. |
| مزودا TTS المحليان `neutts` و `kittentts` | مؤجل | مزودا TTS المحليان/offline غير منفذين. Edge TTS منفذ بشكل منفصل كمزود شبكي. |
| Mistral TTS/STT | مؤجل | غير منفذ. |
| auto-TTS في البوابة | `voice.autoTts: true` | تُطبق حدود لكل رد ولكل ساعة. |
| مزودو المتصفح السحابي المؤجلون | `browser.backend` يقبل أسماء قديمة | خلفيات Firecrawl وCamofox للمتصفح تُبلغ حالة غير متاحة. browser-use يبقى مزودًا سحابيًا مؤجلًا. |
| مزودو بحث الويب المؤجلون | `web.backend` يقبل أسماء وهمية | Firecrawl وParallel وTavily وExa وSearXNG يُبلغون غير متاحين حتى عند التكوين. يتطلب DDGS إعداد Python مُدارًا صريحًا ولا يثبت أثناء بحث وقت التشغيل. |

## موجود لكن غير مثبت عمليًا

هذه القنوات والمزودات موجودة في الكود لكنها تفتقر لأدلة تحقق حية لـ v0.1.0.

| السطح | الدليل | الملاحظة |
|-------|--------|---------|
| قناة Discord | تجريبية | المحول موجود؛ التحقق الحي غير مكتمل. |
| مزود Google LLM | قابل للتكوين/معروف بالكتالوج | مسار التكوين موجود لكنه غير مثبت عمليًا في هذا البناء. |
| مزود Anthropic LLM | معروف بالكتالوج | توجد بياناته الوصفية ومدخلاته في الكتالوج، لكنه غير ظاهر كمسار إعداد/اختيار نموذج وغير قابل للتشغيل في هذا البناء. |

## قيود وقت التشغيل المعروفة

- `doctor --live` قد ينجح مع نص استجابة `[empty]` لبعض المزودين.
- OpenRouter يعمل في وقت التشغيل لكنه قد يفوت فحوصات دقة المحتوى.
- دعم نقطة النهاية المحلية / الخاصة المتوافقة مع OpenAI منفذ، لكنه غير مثبت حيًا في هذه البيئة.
- MCP stdio مثبت عمليًا؛ HTTP وتغطية الجهات الخارجية الأوسع تحتاج تحقق المشغل.
- ميكانيكيات ثقة مساحة عمل MCP خشنة الحبيبات.
- عرض الذاكرة انتقائي لكنه غير مرتب. لا يوجد معالجة للحداثة/القدم.
- على المزودين غير المرئيين، يتدهور تحليل الصورة إلى بيانات وصفية فقط.
- حالة البوابة تُبلّغ الجاهزية، وليس الحيوية الفعلية للعملية في الخلفية.
- تعريب CLI في وقت التشغيل غير مكتمل. عرض العربية في الطرفية يدعم التشكيل والاتجاه ثنائي الاتجاه، لكن ليست كل نصوص CLI مترجمة.
- بيئة التقييم موجودة لكنها ليست معياراً آلياً مقيماً.
- جهاز التحقق الداخلي ألفا يدوي وليس بوابة إصدار صارمة بعد.
- دعم محتوى رسالة المزود تم توسيعه للرؤية، لكن أماكن كثيرة تفترض محتوى نصي مفهومياً.
- منطق المنتج مختلط مع اهتمامات التنسيق/التوصيل في بعض مسارات القنوات.
- اكتشاف قدرة المزود الحية يستحق إشارة مشغّل صريحة.
- استدعاء الجلسة يعمل، لكن إدارة النسب/التاريخ الأكثر ثراء مفقودة.
- تأثيرات الملف الشخصي بالإضافة إلى توجيه المحة تبقى غير كاملة.
- تثبيت npm العالمي يتطلب نشر الحزمة على npm. بيانات الحزمة جاهزة للنشر، لكن النشر يحدث عبر عملية الإصدار.
- تثبيت Homebrew يعتمد على tap خارجي (`KemetResearch/homebrew-tap`) غير تابع لهذا المستودع.
- تثبيت Docker يعتمد على توفر صورة GHCR وتوفير العلامات.
- التخزين الآلي على شجرة عمل متسخة غير منفذ.

## دعم نظام التشغيل

- macOS 11 Big Sur والأحدث هو الحد المعلن.
- Linux: أي توزيعة حديثة مع systemd وglibc. مُتحقق على Ubuntu 22.04+ وDebian 12+.
- WSL2 أفضل جهد. مسارات الصوت/الميكروفون وخدمات مستخدم systemd لها حالات حافة معروفة.
- Termux أفضل جهد. المثبت يحل تخطيط Termux لكنه ليس هدف تحقق أساسي.
- Windows الأصلي غير مدعوم.

## ماذا يعني هذا

المزود المسجل ليس مزودًا عاملًا. القناة الموجودة ليست قناة مُتحققة. الميزة التجريبية ليست ضمان إطلاق. راجع هذه الصفحة قبل الإبلاغ عن خطأ؛ قد يكون الخطأ قيدًا موثقًا.
