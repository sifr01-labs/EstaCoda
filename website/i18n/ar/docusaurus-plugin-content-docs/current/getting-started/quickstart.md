# البداية السريعة

يوصلك هذا الدليل من جهاز نظيف إلى جلسة محلية متحقَّق منها. ثبّت `EstaCoda`، شغّل الإعداد الأول، اضبط مزوّدًا ونموذجًا واحدًا يعملان، تحقق من الجاهزية، ثم ابدأ جلسة `CLI` محلية قبل إضافة القنوات أو الصوت أو المتصفح أو توليد الصور أو أي سطح تشغيل آخر.

## لمن هذه الصفحة

استخدم هذه الصفحة إذا كنت:

- تثبّت `EstaCoda` لأول مرة
- تضبط مزوّدًا ونموذجًا واحدًا
- تتحقق من `CLI` قبل تفعيل `Telegram` أو `WhatsApp` أو الصوت أو المتصفح أو توليد الصور
- تصلح تثبيتًا اكتمل لكنه لا ينتج جلسة تعمل
- تجهّز نسخة مصدرية للتطوير

## أسرع مسار

اختر الصف الذي يطابق هدفك:

| الهدف | ابدأ من هنا | ثم |
|---|---|---|
| أريد تشغيل `EstaCoda` محليًا | ثبّت، ثم شغّل `estacoda setup` | اقبل خيار التشغيل النهائي أو شغّل `estacoda` |
| ثبّتّه سابقًا | `estacoda verify` | افحص `estacoda model status` |
| شغّلت `estacoda` وظهر الإعداد | أكمل مسار الإعداد | استخدم `estacoda setup` إذا خرجت مبكرًا |
| أريد `Telegram` أو `WhatsApp` | احصل أولًا على جلسة `CLI` محلية تعمل | اضبط القنوات بعد التحقق |
| أريد الصوت أو بحث الويب أو المتصفح أو توليد الصور | أنهِ الإعداد الأساسي أولًا | أضف القدرات الاختيارية واحدة تلو الأخرى |
| أريد تعديل المصدر | استنسخ المستودع | شغّل `./scripts/setup-estacoda.sh` |
| هناك شيء معطّل | `estacoda doctor` | أصلح أول مانع قبل إضافة ميزات أخرى |

إذا لم تعمل جلسة `CLI` محلية عادية، لا تفعّل القنوات أو الصوت أو المتصفح أو توليد الصور أو أسطح التطور بعد. احصل أولًا على تفاعل محلي نظيف واحد.

---

## 1. ثبّت `EstaCoda`

مسار التثبيت الافتراضي يستخدم المثبّت العام:

```bash
curl -fsSL https://estacoda.kemetresearch.com/install.sh | bash
```

ينشئ هذا تثبيتًا مُدار المصدر تحت `~/.estacoda/estacoda`، ثم يبني المشروع، ويكتب ملف تشغيل إلى `~/.local/bin/estacoda`، ويشغّل `estacoda init`.

إذا لم يكن `~/.local/bin` ضمن `PATH`، أضفه:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

## 2. شغّل الإعداد الأول

ابدأ مسار الإعداد المُراجع:

```bash
estacoda setup
```

إذا شغّلت `estacoda` قبل اكتمال الإعداد، يكتشف `EstaCoda` الحالة غير المكتملة ويوجهك إلى الإعداد.

يمر معالج الإعداد الأول بهذه المراحل:

| الخطوة | ماذا تفعل | النتيجة |
|---|---|---|
| اللغة ونمط الواجهة | يختار لغة الإعداد ونمط الطرفية | يستخدم الإعداد نمط الواجهة المختار |
| مساحة العمل | يختار مسار مساحة العمل | يعرف `EstaCoda` أين يمكنه العمل |
| الثقة بمساحة العمل | يحدد الثقة للمجلد المختار | يتبع سلوك الأدوات قرار الثقة |
| نقطة نهاية المزود والنموذج | يختار إعداد المزوّد والنموذج الأساسي، بما في ذلك عنوان النقطة المحلية/الخاصة عند الحاجة | يصبح لديك إعداد نموذج قابل للاستخدام |
| معالجة الاعتماد | يخزّن الاعتماد أو يعيد استخدامه أو يتجاوزه حسب طريقة اعتماد المزود | لا تظهر الأسرار الخام في شاشة المراجعة |
| وضع الأمان | يختار سلوك الموافقات | تتطلب الأفعال الحساسة مستوى الموافقة المحدد |
| `Agent Evolution` | يختار سلوك التعلم والمراجعة | يتبع سلوك التطور إعدادات المراجعة |
| القدرات الاختيارية | يضبط القنوات والصوت وبحث الويب والمتصفح | تُضاف الأسطح الإضافية فقط عند اختيارها |
| المراجعة والتطبيق | يعرض ملخصًا نهائيًا قبل الكتابة | تُطبّق التغييرات بعد التأكيد |
| التحقق والتشغيل | يفحص الجاهزية ويعرض تشغيل الجلسة | تصل إلى جلسة محلية |

الإعداد مُراجع قبل التطبيق. يعرض المعالج ملخصًا قبل حفظ تغييرات الإعداد، ولا يعرض الأسرار الخام في مخرجات المراجعة.

يعرض المعالج خيار `رجوع` في مطالبات الاختيار عندما يساعد على تصحيح اختيار سابق. تحتفظ مطالبات النص والأسرار بسلوكها المعتاد.

يمكن لمسار الإعداد الأول ضبط القنوات والصوت وبحث الويب والمتصفح. توليد الصور، وإعدادات النماذج الاحتياطية، والإعدادات الأعمق للمزوّدين متاحة بعد الإعداد الأساسي من خلال محرّر الإعدادات وأوامر `CLI` المخصصة.

## 3. اضبط نموذجًا واحدًا يعمل

اختر مزوّدًا ونموذجًا واحدًا يعملان أولًا. للخوادم المحلية أو الخاصة المتوافقة مع OpenAI مثل Ollama أو LM Studio أو llama.cpp أو vLLM، اختر مسار نقطة النهاية المحلية / الخاصة وأدخل عنوان نقطة النهاية عندما يُطلب منك ذلك. مفاتيح API اختيارية لهذا المسار. لا تضبط النماذج الاحتياطية أو القنوات أو القدرات الاختيارية قبل أن يتحقق الإعداد الأساسي.

بعد الإعداد، افحص إعداد النموذج النشط:

```bash
estacoda model status
```

تُكتب حالة الملف الشخصي تحت:

```text
~/.estacoda/profiles/default/
```

## 4. ابدأ أول جلسة

إذا عرض معالج الإعداد تشغيل `EstaCoda`، اقبل ذلك. وإلا شغّل:

```bash
estacoda
```

استخدم `prompt` يسهل التحقق منه:

```text
Check this directory and tell me what kind of project this is.
Then list the first three files you would inspect next.
```

الجلسة الأولى التي تعمل تعني:

| الفحص | شكل النجاح |
|---|---|
| بدء التشغيل | يبدأ `EstaCoda` بلا أخطاء مزوّد أو نموذج |
| إعداد النموذج | يعرض `estacoda model status` الإعداد المختار |
| أول إجابة | يرد الوكيل بشكل طبيعي |
| الدور الثاني | يعمل سؤال متابعة داخل الجلسة نفسها |
| الأدوات | يعرض `/tools` الأدوات المتاحة أو يشرح سبب عدم توفرها |
| الأمان | تشرح الموافقات أو الرفض ما حدث |

## 5. تحقق من الإعداد

شغّل هذه الأوامر بعد الجلسة الأولى:

```bash
estacoda verify
estacoda model status
estacoda doctor
```

استخدم `estacoda doctor --live` فقط عندما يبدو الإعداد المحلي صحيحًا لكن اتصال المزوّد ما زال يفشل.

## 6. جرّب عناصر التحكم الأساسية

داخل الجلسة:

| الأمر | الاستخدام |
|---|---|
| `/help` | عرض أوامر الجلسة المتاحة |
| `/tools` | عرض الأدوات المتاحة |
| `/model` | فحص أو تغيير إعداد النموذج للجلسة |
| `/status` | عرض حالة الجلسة والتشغيل |
| `/interrupt` | إلغاء دور نشط |
| `/exit` | الخروج من الجلسة |

اكتب `/` لفتح قائمة الأوامر. استخدم `/help` داخل الجلسة لمعرفة عناصر التحكم المتاحة في النسخة المثبتة لديك.

أضف الأسطح الاختيارية فقط بعد أن يعمل `CLI` المحلي.

| الطبقة التالية | استخدمها عندما | ابدأ من |
|---|---|---|
| `Telegram` | تريد وصولًا عن بُعد من `Telegram` | [القنوات](../user-guide/channels.md) |
| `WhatsApp` | تريد وصول `WhatsApp` مرتبطًا عبر `QR` | [القنوات](../user-guide/channels.md) |
| الصوت | تريد `STT/TTS` | [الصوت](../user-guide/voice.md) |
| بحث الويب | تريد بحث ويب حيًا عبر `Brave` أو `DDGS` | [مرجع المزودين](../reference/provider-reference.md) |
| المتصفح | تريد عملًا مدعومًا بالمتصفح | [المتصفح](../user-guide/browser.md) |
| توليد الصور | تريد أدوات صور مدعومة بمزوّد | [توليد الصور](../user-guide/image-generation.md) |

| `MCP` | تريد خوادم أدوات خارجية | [الأدوات](../user-guide/tools.md) |

## 8. أعطال شائعة

| العرض | السبب المرجح | أول إصلاح |
|---|---|---|
| يفتح `estacoda` الإعداد بدل الجلسة | الإعداد غير مكتمل | شغّل `estacoda setup` |
| يكتمل الإعداد لكن التشغيل محجوب | تم تأجيل الثقة بمساحة العمل | أعد تشغيل الإعداد أو وثّق مساحة العمل |
| إعداد النموذج موجود لكنه لا يجيب | اعتماد مفقود أو غير صالح | شغّل `estacoda model status` ثم `estacoda setup` |
| الأوامر غير موجودة بعد التثبيت | `~/.local/bin` ليس ضمن `PATH` | أضفه إلى ملف `shell` وأعد التحميل |
| الأدوات غير متاحة | سياسة الأدوات أو إعداد الملف الشخصي لا يعرّضها | شغّل `/tools` و `estacoda verify` |
| رسائل القنوات لا تصل | إعداد المصادقة أو قائمة السماح أو المنصة غير مكتمل | راجع وثائق القنوات والتشخيص |
| تغيّر السلوك بعد تعديلات | تغيّر إعداد الملف الشخصي أو حالة الثقة | شغّل `estacoda doctor` قبل تغيير إعدادات أخرى |

## 9. ترتيب التعافي

استخدم هذا الترتيب قبل التصحيح اليدوي:

```bash
estacoda verify
estacoda model status
estacoda doctor
estacoda setup
```

استخدم `estacoda doctor --live` بعد ذلك فقط إذا بقي اتصال المزوّد هو سبب الفشل المرجح.

## 10. مسارات التثبيت المتقدمة

ثبّت في مجلد مخصص وتجاوز تهيئة الحالة الأولى:

```bash
curl -fsSL https://estacoda.kemetresearch.com/install.sh | bash -s -- --dir <path> --skip-init
```

إذا كنت تنوي تعديل المصدر، استنسخ المستودع وشغّل سكربت الإعداد:

```bash
git clone https://github.com/KemetResearch/EstaCoda.git
cd EstaCoda
./scripts/setup-estacoda.sh
```

ينشئ هذا تثبيتًا يدوي المصدر. تبقى نسخة المصدر محفوظة عند إلغاء التثبيت، ويعمل التحديث في وضع الفحص والإرشاد.

لكل مسارات التثبيت، راجع [التثبيت](./installation.md).

## 11. مرجع سريع

| الأمر | الاستخدام |
|---|---|
| `estacoda` | بدء جلسة تفاعلية، أو التوجيه إلى الإعداد إذا كان غير مكتمل |
| `estacoda setup` | تشغيل الإعداد أو الإصلاح المُراجع |
| `estacoda verify` | فحص جاهزية الإعداد |
| `estacoda model status` | فحص إعدادات المزوّد/النموذج النشطة |
| `estacoda doctor` | تقرير الصحة والإصلاحات المطلوبة |
| `estacoda doctor --live` | إضافة فحوصات اتصال حية للمزوّد |
| `estacoda --help` | عرض مساعدة `CLI` |
